O pkg_build copia os headers para:

DESTDIR/usr/include/...


O admV2 empacota esse DESTDIR em um tarball com paths como:

usr/include/linux/*.h


Na hora de instalar o pacote:

ADM_ROOTFS=/opt/systems/glibc-root ./admV2.sh install linux-6.17.9-api-headers

o admV2 extrai o tarball em ADM_ROOTFS, então:

os headers vão para:
/opt/systems/glibc-root/usr/include/{linux,asm,asm-generic,...}

Mesma lógica vale para musl:

source ./profile-musl.sh
ADM_ROOTFS=/opt/systems/musl-root \
  ./admV2.sh build   linux-6.17.9-api-headers
ADM_ROOTFS=/opt/systems/musl-root \
  ./admV2.sh install linux-6.17.9-api-headers
